## Development Rules

- Preserve existing architecture unless asked to refactor.
- Use TypeScript.
- Use accessible HTML.
- Keep components clean and reusable.
- Avoid unnecessary dependencies.
- Use existing project styles before introducing new visual patterns.
- Test after meaningful changes.
- Explain changed files and commands run.

## Design Rules

- Follow DESIGN.md.
- Follow COMPONENT_SOURCES.md.
- Follow SECURITY_RULES.md.
- Prioritize mobile responsiveness.
- Avoid generic AI-generated UI patterns.

## Before Adding Packages

- Explain why the package is needed.
- Check whether native code or existing components can solve it.
- Prefer small, well-maintained packages.
- Avoid packages that create dependency bloat.
